《疯狂SpringCould》 Ribbon

一、简介

  • Ribbon是Netflix下的负载均衡项目，在集群中为各个客户端的通信提供了支持。主要实现中间层应用程序的负载均衡。

  • Ribbon可以与Eureka整合使用，同样被集成到SpringCould中，使用者可以使用SpringCouldAPI或Ribbon原生API实现负载均衡

1.特性

  • 负载均衡器：支持拔插式的负载均衡规则

  • 对多种协议提供支持，如HTTP、TCP、UDP

  • 集成了负载均衡功能的客户端

2.子模块

  • ribbon-core：核心，包括负载均衡器接口定义、客户端接口定义、内置负载均衡实现等API

  • ribbon-eureka：为Eureka客户端提供的负载均衡实现类

  • ribbon-httpclient：对Apache的HttpClient进行封装，提供含有负载均衡功能的REST客户端

3.负载均衡器组件

Ribbon中的负载均衡器主要与集群中的各个服务器进行通信，根据特定逻辑在服务器列表中循环，维护服务器的IP、DNS名称等信息

负载均衡器三大子模块：

  • Rule：一个逻辑组件，决定从服务器列表中返回哪个服务器实例

  • Ping：使用定时器确保服务器网路可连接

  • ServerList：服务器列表，通过静态的配置确定负载的服务器，也可动态指定服务器列表。

    • 若动态指定服务器列表，会有后台的线程来刷新该列表

三、Ribbon负载均衡机制

Ribbon提供了几个负载均衡的组件，主要进行服务器选择，让请求转给合适的服务器处理

1.使用负载均衡器选择服务器

public static void main(String[] args) {
    // 创建负载均衡器
    ILoadBalancer lb = new BaseLoadBalancer();
    // 添加服务器
    List<Server> servers = new ArrayList<>();
    servers.add(new Server(";localhost",8080));
    servers.add(new Server(";localhost",8081));
    lb.addServers(servers);
    // 进行6次服务器选择
    for(int i=0; i<6; i++){
        Server s = lb.chooseServer(null);
        System.out.println(s);
    }
}

3.Ribbon自带的Rule

  • RoundRobinRule：默认，通过简单的轮询服务列表来选择

  • AvailabilityFilteringRule：该规则会忽略以下服务器

    • 无法连接的服务器：默认状态下，3次连接失败，该服务器会被置为短路状态30s；如果再连接失败，短路状态持续时间会以几何级数增加。可通过修改niws.loadbalancer.<clientName>.connectionFailureCountThreshold属性配置连接失败次数

    • 并发过高的服务器：连接并发数过高就忽略。可通过修改<clientName>.ribbon.ActiveConnectionsLimit属性来设置最高并发数

  • WeightedResponseTimeRule：为每个服务器赋予一个权重值。服务器响应时间越长，权重值就越少

  • ZoneAvoidanceRule：以区域、可用服务器为基础进行选择，使用Zone对服务器进行分类，可理解为机房

  • BestAvailableRule：忽略短路的服务器，选择并发数低的服务器

  • RandomRule：随机选择可用的服务器

  • RetryRule：若使用RoundRobinRule选择的服务器法连接

4.Ping机制

负载均衡器中，每隔一段时间，会去Ping服务器，判断服务器是否存活。

该工作由IPing接口的实现类负责，默认为DummyPing

单独使用Ribbon时，默认不会激活Ping机制

lb.addServers(servers);

Thread.sleep(6000);
for(Server s: lb.getAllServers()){
    System.out.println(s.getHostPort() + "状态为：" + s.isAlive());
}

五、RestTemplate负载均衡

1.@LoadBalanced注解

RestTemplate本是SpringWeb项目中的一个REST客户端，遵循REST设计原则，提供简单的API让我们调用HTTP服务，本身不带有负载均衡功能。

在SpringCould中，Spring容器启动时，会对@LoadBalanced注解修饰的RestTemplate添加拦截器，拦截器中使用了Spring封装的负载均衡客户端LoadBalancerClient来处理请求，因此RestTemplate就看似有了负载均衡功能。

2.自定义注解和拦截器

@Target({ElementType.FIELD,ElementType.PARAMETER,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Qualifier
public @interface MyLoadBalanced {
}
public class MyHttpInterceptor implements ClientHttpRequestInterceptor{
    @Override
    public ClientHttpResponse intercept(HttpRequest request, byte[] body, ClientHttpRequestExecution execution) throws IOException {
        // 自定义拦截器实现
        System.out.println("原来的URI：" + request.getURI());
        MyHttpRequest newRequest = new MyHttpRequest(request);
        System.out.println("拦截后的URI：" + newRequest.getURI());
        return execution.execute(newRequest, body);
    }
}
