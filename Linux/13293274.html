Linux （鸟哥）Bash

一、硬件、内核、Shell

用户通过shell操纵应用程序的接口与内核沟通，内核控制硬件完成工作。

Linux是C语言编写的，因此Cshell热门

系统合法shell存放于/etc/shells文件中；用户预设shell存放于/etc/passwd文件中

bash优点

  • 记忆功能（history）：~/.bash_history（注销后写入）

  • 补全功能（TAB）

  • 别名功能（alias）

  • 工作前/背景控制（&）

  • 程序化脚本（shell scripts）

  • 通配符（*等）

二、变量

  • 变量在取用时，前面需要加上$或${}

  • 将变量变成环境变量（子程序）用export；取消变量用unset

  • env/export列出环境变量；set/declare列出环境变量和自定变量

  • 双引号中的特殊字符，如$，可以保留原有特性，而单引号为纯文本

  • 跳脱字符\将特殊符号（如Enter、$、\、'等）变成一般字符

  • 可用`指令`或$(指令)将指令结果赋予变量

  • 别名：别名='指令 选项'；取消别名用unalias

  • 常见变量：PS1：提示字符；$：该shell的PID；?：上一个指令的返回值（0为执行成功）；MACHTYPE：机器等级

变量替换

   变量删除只能从开始或末尾删

 

三、Bash操作环境

1.路径与指令搜寻顺序

指令执行顺序：

  • 相对/绝对路径执行。如“/bin/ls”

  • “alias”找到的指令执行

  • bash内建的builtin执行

  • 按“$PATH”搜索到的执行

查看指令执行顺序：type -a ls

2.Bash进站欢迎讯息

/etc/issue和/etc/issue.net和/etc/motd

3.Bash的环境配置文件

login shell：要求输入用户的账号和密码。读取/etc/profile和~/.bash_profile

non-login shell：无需重复登录。如右键open terminal或bash子程序。读取~/.bashrc

  • /etc/profile：所有登陆者都读。文件也呼叫外部文件，如/etc/profile.d/*.sh

  • ~/.bash_profile：个人配置，若没有就读~/.bash_login，再没有就读~/.profile。文件也呼叫外部文件，如~/.bashrc

  • ~/.bashrc：PS1，呼叫/etc/bashrc

source或.：读取环境配置文件到现有环境

其他配置文件

  • /etc/man_db.conf：使用man时，man page的路径

  • ~/.bash_history：历史命令读入内存

  • ~/.bash_logout：注销Bash后，系统做什么

四、命令操作

1.命令通配符

命令说明
* 0到无数个任意字符
? 一定有一个任意字符
[] 一定有一个在括号内的字符
[^] 非括号内字符就行

2.特殊符号

命令说明
# 注释
\ 跳脱，转义
| 管线
; 连续指令
~ 家目录
$ 变量前置符
& 指令后台工作
! 非
/ 路径
>, >> 输出：取代/累加
<, << 输入
'' 纯文本
"" 保留"$"功能
`` 先执行的指令，或$()
() 子shell
{} 命令区块

3.组合键

命令说明
[Ctrl]+d EOF
[Ctrl]+m Enter
[Ctrl]+s 暂停屏幕输出
[Ctrl]+q 恢复屏幕输出
[Ctrl]+z 暂停目前命令
[Ctrl]+c 终止目前命令
[Ctrl]+u, [Ctrl]+k 删除光标之前/后的指令
[Home], [End] 光标移动到指令最前/后面
[Ctrl]+a, [Ctrl]+e 同上

4.执行顺序

 cmd;cmd  #前面执行完就执行后面
 cmd&&cmd  #前面执行完正确才执行后面，错误不执行
 cmd||cmd  #前面执行完正确后面就不执行，错误才执行

五、数据流重导向

  • >为覆盖；>>为累加

  • /dev/null：错误信息垃圾桶

# ×：两股数据会交叉写进文件内，次序错乱
find /home -name xxx > list 2>list
# √：两股信息写入同一文件
find /home -name xxx > list 2>&1
find /home -name xxx &> list
# find /home -name xxx 2>/dev/null 1>&2

【<的用法：P468】

六、指令合集

1.基本指令

①type

查询是否是bash内建指令

②echo

常用的输出指令

③read

将键盘输入作为变量值

④declare

声明变量及类型

⑤history

历史命令

⑥ulimit

限制用户操作

⑦split

文件分区

【注】-表示传入的数据流（文件）

2.管线命令

仅处理正确信息，后面的指令需要能接受STDIN的数据

①提取：cut/grep

cut：以行为单位，取出想要的字段。但处理多空格的数据时吃力

grep：以行为单位，取出想要的行。

②排序：sort/uniq/wc

sort：根据不同数据形态排序

uniq：重复的仅列一个。但需配合排序

wc：

③双向重导向：tee

tee：同时将数据流分送到文件和屏幕

④字符转换：tr/col/expand/join/paste

tr：删除/替换一段信息中的文字

col：Tab转空格

expand：Tab转空格

join：两个文件中相同字段行放在一起，但需先排序处理

paste：直接将两行放一起，中间以Tab隔开

⑤参数代换：xargs

  • 许多指令并不支持管线，因此可用xargs提供该指令使用STDIN

  • 读入STDIN的数据，以空格符或断行符分割成args。空格容易误判

id只能接受一个参数，且不是管线命令

3.环境设定

①set

设定整个指令输入/输出的环境

②stty

设定终端机输入按键的作用

 其他

bash进行数值计算：$((算式))（仅支持整数数据）
